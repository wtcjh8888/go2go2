# Go2 3D 导航系统方案

## 1. 背景

原导航系统将 FAST-LIO 建图输出的 3D 点云投影为 2D 栅格地图，再用 A* 做全局规划。投影过程丢失了高度信息，无法处理坡道、楼梯等 3D 地形。

参考山猫 M20 的方案，改造为直接在 3D 点云上做图搜索规划，保留完整的高度信息。

## 2. 整体架构

```
PCD 点云文件 (FAST-LIO 建图输出)
  │
  ▼
┌──────────────┐
│  map_server  │──→ /nav_graph_cloud (PointCloud2, 关键点+可通行标志)
└──────────────┘
       │
       ▼
┌───────────────────────┐
│ odometry_to_base_link │←── /Odometry + TF(body → base_link)
│                       │──→ /Odometry_base_link
└───────────────────────┘
       │
       ▼
┌──────────────┐
│ path_planner │←── /nav_graph_cloud + /Odometry_base_link + /goal_pose
│              │──→ /planned_path (Path, 每个点带 XYZ)
└──────────────┘
       │
       ▼
┌──────────────────┐
│ motion_controller│←── /planned_path + /Odometry_base_link
│                  │──→ /cmd_vel_raw (Twist, 坡度调速)
└──────────────────┘
       │
       ▼
┌──────────────────┐
│ obstacle_avoider │←── /cmd_vel_raw + /cloud_registered_body + /Odometry_base_link
│                  │──→ /cmd_vel (Twist, 3D 避障后)
└──────────────────┘
       │
       ▼
┌──────────────┐
│ webrtc_bridge│──→ Go2 WebRTC (机器人执行)
└──────────────┘
```

**数据流说明：**

| 阶段 | 输入 | 输出 | 说明 |
|------|------|------|------|
| 地图构建 | PCD 文件 | `/nav_graph_cloud` | 体素化→关键点→KD-tree 建图，PointCloud2 发布 |
| 位姿转换 | `/Odometry` + `body -> base_link` TF | `/Odometry_base_link` | 将 FAST-LIO 的雷达/IMU body 位姿转换为机器人底盘 base_link 位姿 |
| 全局规划 | 导航图 + 里程计 + 目标 | `/planned_path` | Dijkstra 搜索 + 插值 + 平滑 |
| 运动控制 | 路径 + 里程计 | `/cmd_vel_raw` | XY 平面 Pure Pursuit 跟踪 |
| 局部避障 | 原始速度 + 实时点云 + 里程计 | `/cmd_vel` | 3D 高度过滤 + VFH 避障 |

### 2.1 Odometry 坐标约定

FAST-LIO 原始 `/Odometry` 的 `child_frame_id` 对应雷达/IMU 的 `body`。因为 MID360S 倾斜安装，`body` 的坐标轴和 RViz 中的红色 odometry 箭头看起来会向下倾斜，这是算法的真实输出，不建议直接改掉或伪装成水平底盘。

导航需要的是机器人底盘 `base_link` 位姿，因此新增 `odometry_to_base_link` 节点：

```text
/Odometry(body) + TF(body -> base_link)
  -> /Odometry_base_link(base_link)
```

后续导航模块统一订阅 `/Odometry_base_link`。这样既保留 FAST-LIO 原始输出用于调试，又保证路径规划、避障和运动控制使用水平底盘坐标。

### 2.2 当前楼层地面提取

当前使用的地图文件是 `/home/w/C206Go2/src/FAST_LIO-ROS2/PCD/scan.pcd`。这张地图是在二楼扫描得到的，点云里可能混入其他楼层的点，因此导航不能简单把全局点云中最密集的 Z 层当作可通行地面。

Go2 站立时 `base_link` 位于身体附近，约高于地面 0.30m。导航启动时 `map_server` 会优先等待 `/Odometry_base_link`，并用下面的关系锁定当前楼层：

```text
floor_z = base_link_z - robot_base_height
```

默认参数：

```yaml
floor_detection_mode: "odom"
robot_base_height: 0.30
ground_band_below: 0.20
ground_band_above: 0.25
```

也就是说，先根据机器人当前所在楼层估计地面高度，再只从这个高度附近提取导航关键点。若启动时拿不到 `/Odometry_base_link`，才回退到点云 Z 分布自动检测。

## 3. 核心算法

### 3.1 地图构建 (`map_server.py`)

```
PCD 点云
  → 当前楼层地面高度估计 (base_link_z - 0.30m)
  → 高度过滤 (ground_z_min ~ ground_z_max)
  → 体素化 (voxel_resolution=0.2m, 每体素至少 3 点)
  → 关键点提取 (keypoint_resolution=0.2m 降采样)
  → KD-tree 建图 (connection_radius=0.5m, 高差≤0.2m, 坡度≤45°)
  → PointCloud2 发布
```

**体素化算法：**
- 将点云坐标除以分辨率取整，得到整数体素坐标
- `np.unique` 按体素分组，过滤点数不足的体素
- 每个有效体素取质心作为中心，Z 取均值作为高度

**图构建算法：**
- 用 `scipy.spatial.cKDTree` 建立空间索引
- 对每个关键点，`query_ball_point` 查找半径内邻居
- 过滤条件：高差 ≤ `max_height_change`，坡度 ≤ `max_slope_angle`
- 边权 = 欧氏距离 × (1 + 4 × 高差/max_height_change)，高差越大代价越高

### 3.2 路径规划 (`path_planner.py`)

```
收到 /nav_graph_cloud → 解析 PointCloud2 → build_nav_graph 重建图
收到 /goal_pose (XYZ) → find_nearest_keypoint → dijkstra_search → graph_path_to_world → smooth_path_3d → 发布 /planned_path
```

**Dijkstra 搜索：**
- 标准堆优化 Dijkstra，在邻接图上搜索最短路径
- 返回关键点索引序列

**路径插值 (`graph_path_to_world`)：**
- 相邻关键点之间插入 N 个等距中间点（默认 5 个）
- 使路径点更密集，跟踪更平滑

**路径平滑 (`smooth_path_3d`)：**
- 移动平均平滑：XY 方向全权重，Z 方向降低权重
- `height_weight` 参数控制 Z 方向平滑程度（0=不平滑 Z，1=完全平滑）
- 目的：XY 去锯齿，但保留坡道等垂直结构

### 3.3 运动控制 (`motion_controller.py`)

```
收到 /planned_path → 存储路径 (x, y, z) 序列
定时 10Hz:
  → 找最近路径点 (XY 平面距离)
  → 检查目标到达 (XY 平面距离 < goal_tolerance)
  → find_look_ahead_point (XY 平面距离 >= look_ahead)
  → pure_pursuit_control (XY 平面控制)
  → smooth_velocity (加速度限制)
  → 发布 /cmd_vel_raw
```

**Pure Pursuit 控制律：**
- 将目标点转换到机器人坐标系
- 曲率 = 2 × local_y / dist²
- 线速度 = max_lin × cos(角度差)
- 角速度 = clamp(曲率 × vx × 2, ±max_ang)

**Z 的用途：**
- 路径仍保留 z 值，用于显示 3D 地图上的地面高度。
- 控制只使用 XY 平面距离和 yaw，适合当前“基于 3D 点云地图做平面导航”的目标。
- `slope_speed_factor` 默认设为 0.0，避免楼层点云高度噪声影响平面运动控制。

**速度平滑：**
- `smooth_velocity(target, current, dt, max_acc)` 限制加速度
- 防止速度突变对机器人造成冲击

### 3.4 局部避障 (`obstacle_avoider.py`)

```
收到 /cloud_registered_body → 存储点云
收到 /Odometry_base_link → 记录 robot_z
收到 /cmd_vel_raw → 存储原始速度
定时 20Hz:
  → classify_terrain (坡道检测)
     → 坡道？只做速度衰减，不避障
  → check_danger_ahead_3d (高度过滤 + 危险检测)
     → 紧急停止？发布零速度
  → filter_fov_points_3d (高度带 + FOV + 距离过滤)
  → build_vfh_histogram (72 扇区，5° 分辨率)
  → find_best_direction (±90° 内最通畅方向)
  → 正前方通畅？发布原始速度 : compute_avoidance_cmd 计算避障速度
  → 发布 /cmd_vel
```

**3D 高度过滤：**
- 只关注 `[robot_z - ground_clearance, robot_z + height_band]` 范围内的点
- 忽略地面以下的点（地面间隙）和头顶以上的点
- 使避障器区分地面、障碍物、头顶空间

**VFH (Vector Field Histogram)：**
- 360° 分为 72 个扇区（每扇区 5°）
- 每个扇区记录最近障碍物距离
- 在 ±90° 范围内找距离最大的扇区作为最佳通行方向
- 正前方通畅则保持原始速度，否则计算避障转向

**地形分类 (`classify_terrain`)：**
- 对前方点云做最小二乘平面拟合 z = a·x + b
- 计算坡度角 = atan(|a|)
- 坡道（坡度 > threshold）：跳过 VFH 避障，只做速度衰减（坡度越大减速越多）
- 平地：正常执行 VFH 避障流程

## 4. 关键数据结构

### NavGraph

```python
@dataclass
class NavGraph:
    keypoints: np.ndarray       # (N, 3) float32, 关键点 XYZ 坐标
    heights: np.ndarray         # (N,) float32, 每个关键点的平均体素高度
    traversable: np.ndarray     # (N,) bool, 是否可通行
    tree: cKDTree               # scipy.spatial.cKDTree, 空间索引
    adjacency: Dict[int, List[Tuple[int, float]]]  # 邻接表 {i: [(j, cost), ...]}
```

### 导航图发布格式 (PointCloud2)

| 字段 | 偏移 | 类型 | 说明 |
|------|------|------|------|
| x | 0 | float32 | 关键点 X 坐标 |
| y | 4 | float32 | 关键点 Y 坐标 |
| z | 8 | float32 | 关键点 Z 坐标 |
| intensity | 12 | float32 | 可通行标志：1.0=可通行, 0.0=不可通行 |

point_step = 16 bytes, 使用 TRANSIENT_LOCAL QoS 保证晚订阅也能收到。

## 6. 参数配置

### 地图构建参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| voxel_resolution | 0.2 m | 体素边长，越小精度越高但计算量越大 |
| min_points_per_voxel | 3 | 体素内最少点数，过滤稀疏噪声 |
| keypoint_resolution | 0.2 m | 关键点降采样间距 |
| connection_radius | 0.5 m | 关键点最大连接半径 |
| max_height_change | 0.2 m | 相邻关键点最大允许高差 |
| max_slope_angle | 45.0° | 相邻关键点最大允许坡度 |
| ground_z_min/max | 0.0 / 0.5 m | 地面点高度范围 |

### 路径规划参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| planner_frequency | 1.0 Hz | 规划频率 |
| goal_tolerance | 0.3 m | 到达目标的 3D 距离容差 |
| smooth_iterations | 3 | 路径平滑迭代次数 |
| graph_interp_points | 5 | 每段边的插值点数 |

### 运动控制参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| look_ahead_distance | 0.5 m | Pure Pursuit 前视距离 |
| control_frequency | 10.0 Hz | 控制频率 |
| max_linear_speed | 0.5 m/s | 最大线速度 |
| max_angular_speed | 1.0 rad/s | 最大角速度 |
| max_acceleration | 0.5 m/s² | 最大加速度 |
| max_slope_angle | 30.0° | 预留参数，当前平面导航默认不使用 |
| slope_speed_factor | 0.0 | 当前平面导航关闭坡度减速 |

### 避障参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| detection_range | 3.0 m | 检测范围 |
| danger_distance | 0.3 m | 紧急停止距离 |
| warning_distance | 0.8 m | 警告距离 |
| fov_angle | 120.0° | 前方检测角度 |
| height_band | 0.5 m | 高度带上限（相对机器人 Z） |
| ground_clearance | 0.05 m | 地面间隙 |
| slope_threshold | 30.0° | 坡道判定阈值 |

## 9. 与旧方案对比

| 对比项 | 旧方案 (2D) | 新方案 (3D) |
|--------|------------|------------|
| 地图表示 | 2D OccupancyGrid | 3D PointCloud2 关键点图 |
| 规划算法 | A* 在 2D 栅格上搜索 | Dijkstra 在 3D 关键点图上搜索 |
| 高度信息 | 投影丢失 | 完整保留 |
| 坡道处理 | 不支持 | 坡度感知：图构建过滤 + 运动控制减速 |
| 避障 | 2D 点云过滤 | 3D 高度过滤（区分地面/障碍物/头顶） |
| 障碍物层 | 2D 静态 costmap | 删除（实时点云足够） |
| 内存占用 | 栅格大小固定 | 关键点数量动态，通常更小 |
