# Go2 基于 scan.pcd 的平面导航步骤

本文档只保留当前实际使用的导航命令。导航地图使用：

```bash
/home/w/C206Go2/src/FAST_LIO-ROS2/PCD/scan.pcd
```

导航坐标系使用 `camera_init`。导航节点会自动把 FAST-LIO 的 `/Odometry` 转换为 `/Odometry_base_link`，后续路径规划、避障和运动控制都使用 `/Odometry_base_link`。

## 0. 编译

修改代码或参数后执行一次：

```bash
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
colcon build --packages-select go2_navigation
source install/setup.bash
```

## 1. 启动雷达

新开终端 1：

```bash
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch livox_ros_driver2 msg_MID360s_launch.py
```

确认雷达话题：

```bash
ros2 topic list | grep livox
```

应至少看到：

```bash
/livox/lidar
/livox/imu
```

## 2. 启动 FAST-LIO

新开终端 2：

```bash
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch fast_lio mapping_auto_tf.launch.py
```

说明：

```text
/Odometry            FAST-LIO 原始 body 位姿，雷达倾斜安装时可能向下倾斜
body -> base_link    自动补偿的雷达到机器人底盘 TF
```

启动后让机器人静止 2-3 秒，等 IMU 初始化和倾角估计完成。

## 3. 启动导航

新开终端 3：

```bash
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch go2_navigation navigation.launch.py
```

这个 launch 会启动：

```text
odometry_to_base_link
map_server
path_planner
obstacle_avoider
motion_controller
webrtc_bridge
rviz2
```

导航会发布并使用：

```text
/Odometry_base_link
/map_cloud
/nav_graph_cloud
/planned_path
/cmd_vel_raw
/cmd_vel
```

`map_server` 会优先用 `/Odometry_base_link` 的高度估计当前二楼地面：

```text
floor_z = base_link_z - 0.30m
```

这样可以减少 `scan.pcd` 中其他楼层点云对导航图的影响。

## 4. 在 RViz 中发送目标

`navigation.launch.py` 默认会打开 RViz，并加载导航配置。

检查 RViz：

```text
Fixed Frame = camera_init
```

在 RViz 中确认能看到：

```text
/map_cloud
/nav_graph_cloud
/planned_path
```

然后点击工具栏的 `2D Goal Pose`，在地图上点一个较近目标。

也可以用命令发送目标：

```bash
ros2 topic pub --once /goal_pose geometry_msgs/PoseStamped \
  '{header: {frame_id: "camera_init"}, pose: {position: {x: 0.0, y: 5.0, z: 0.0}, orientation: {w: 1.0}}}'
```
