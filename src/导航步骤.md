# Go2 3D 导航测试步骤

## 测试目标

依次验证 4 个模块，最终让机器人自主导航到目标点。

| 测试 | 验证内容 | 需要启动 | 需要机器人 | 需要雷达 |
|------|---------|---------|-----------|---------|
| 1 | 地图加载 | map_server | 否 | 否 |
| 2 | 路径规划 | map_server + path_planner | 否 | 否 |
| 3 | 机器人移动 | LiDAR + SLAM + 导航全部模块 | 是 | 是 |
| 4 | 实时避障 | LiDAR + SLAM + 导航全部模块 | 是 | 是 |

所有测试前先编译：
```bash
cd ~/C206Go2
colcon build --packages-select go2_navigation
source install/setup.bash
```

## 测试 1：地图加载

**目标：** 验证 PCD 文件能正确加载，生成导航关键点图。

**启动：**
```bash
# 终端 1
ros2 run go2_navigation map_server
```

**看终端输出：**
```
[INFO] 读取 151646 个点                    ← PCD 加载成功
[INFO] 自动检测地面: Z=[2.43, 2.93]m       ← 地面检测成功
[INFO] 地面点: 10121                       ← 有地面点（>0）
[INFO] 导航图: 536 个关键点, 6878 条边      ← 图构建成功
[INFO] 原始地图已发布: 37911 点 (/map_cloud) ← 地图发布成功
```

**用 RViz 验证：**
```bash
# 终端 2
rviz2
```
1. 左上角 Fixed Frame 改为 `map`
2. 左下角 Add → By topic：
   - `/map_cloud` → PointCloud2 — 看到完整 3D 地图
   - `/nav_graph_cloud` → PointCloud2 — 看到地面关键点

**通过标准：** 终端无报错，RViz 能看到地图和关键点。

`Ctrl+C` 关掉，进入测试 2。

## 测试 2：路径规划

**目标：** 验证能从当前位置规划出一条到目标点的路径。

**启动：**
```bash
# 终端 1
ros2 launch go2_navigation navigation.launch.py
```

> 导航启动文件会自动启动 `odometry_to_base_link`，把 FAST-LIO 的 `/Odometry`（body 位姿）转换为 `/Odometry_base_link`（base_link 位姿）；路径规划、避障和运动控制默认使用 `/Odometry_base_link`。

**打开 RViz：**
```bash
# 终端 2
rviz2
```
1. Fixed Frame 改为 `map`
2. Add → By topic：
   - `/map_cloud` → PointCloud2 — 看到完整地图
   - `/nav_graph_cloud` → PointCloud2 — 看到导航关键点
   - `/planned_path` → Path — 看规划路径
3. 点击顶部工具栏 **"2D Goal Pose"** 按钮
4. 在地图上点一个目标位置

**看终端输出（path_planner）：**
```
[INFO] 收到目标: (x.xx, x.xx, x.xx)   ← 收到目标
```

**通过标准：** RViz 中能看到从当前位置到目标的路径线。

**常见问题：**
- `Dijkstra 未找到路径` — 目标点不在导航图覆盖范围内，换个近一点的目标
- 没看到路径 — 确认 path_planner 终端有"收到目标"输出

`Ctrl+C` 关掉，进入测试 3。

## 测试 3：机器人移动

**目标：** 验证机器人能沿规划路径移动并到达目标。

**前提：** 已连接机器人（WebRTC IP: 192.168.12.1），已启动雷达

**启动（共 3 个终端）：**
```bash
# 终端 1：雷达
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch livox_ros_driver2 msg_MID360s_launch.py

# 终端 2：自动补偿 SLAM（提供里程计）
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch fast_lio mapping_auto_tf.launch.py

# 终端 3：导航
cd ~/C206Go2
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch go2_navigation navigation.launch.py
```

> 导航启动文件会自动启动 `odometry_to_base_link`，把 FAST-LIO 的 `/Odometry`（body 位姿）转换为 `/Odometry_base_link`（base_link 位姿）；路径规划、避障和运动控制默认使用 `/Odometry_base_link`。

**打开 RViz：**
```bash
# 终端 4
rviz2
```
1. Fixed Frame 改为 `map`
2. Add → By topic：
   - `/map_cloud` → PointCloud2 — 看到完整地图
   - `/nav_graph_cloud` → PointCloud2 — 看到导航关键点
   - `/planned_path` → Path — 看规划路径
   - `/cloud_registered` → PointCloud2 — 看到实时扫描点云（可选）
3. 点击 **"2D Goal Pose"**，在地图上点目标

**或命令行发送目标：**
```bash
# 终端 5
ros2 topic pub --once /goal_pose geometry_msgs/PoseStamped \
  '{header: {frame_id: "map"}, pose: {position: {x: 0.0, y: 5.0, z: 2.7}, orientation: {w: 1.0}}}'
```

**观察：**
- 机器人开始沿路径移动
- 到达目标附近后自动停止

**监控速度：**
```bash
# 终端 6
ros2 topic echo /cmd_vel
```
- 移动时有非零的 `linear.x` 和 `angular.z`
- 到达目标后输出全零

**通过标准：** 机器人移动到目标位置后停止。

`Ctrl+C` 关掉，进入测试 4。

## 测试 4：实时避障

**目标：** 验证机器人遇到障碍物时能减速或绕行。

**前提：** 测试 3 已通过

**启动方式与测试 3 相同**（LiDAR + SLAM + 导航）

**发送目标（RViz 或命令行）：**
```bash
ros2 topic pub --once /goal_pose geometry_msgs/PoseStamped \
  '{header: {frame_id: "map"}, pose: {position: {x: 0.0, y: 5.0, z: 2.7}, orientation: {w: 1.0}}}'
```

**观察：**
- 机器人开始移动
- 在机器人前方放一个障碍物（如箱子）
- 机器人应减速或转向绕开

**通过标准：** 机器人遇到障碍物时自动减速或绕行。

## 常用命令

```bash
ros2 node list                        # 查看运行中的节点
ros2 topic list                       # 查看所有话题
ros2 topic echo /cmd_vel              # 查看速度指令
ros2 topic echo /planned_path --once  # 查看规划路径
ros2 topic echo /Odometry --once      # 查看 FAST-LIO 原始 body 位姿，雷达倾斜安装时姿态可能向下倾斜
ros2 topic echo /Odometry_base_link --once  # 查看导航使用的 base_link 位姿
ros2 run tf2_ros tf2_echo body base_link     # 查看 body 到 base_link 的安装角补偿 TF
```

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| FileNotFoundError: PCD | PCD 路径错误 | 检查 pcd_path 参数 |
| 地面点为 0 | 自动检测不准 | 手动设置 ground_z_min/max |
| Dijkstra 未找到路径 | 目标太远 | 换个近一点的目标 |
| 收到两次导航图 | 多个 map_server | `ros2 node list` 检查，杀掉多余的 |
| 机器人不动 | webrtc_bridge 没启动 | 检查 WebRTC 连接和 IP |
| 机器人不避障 | 没有实时点云 | 确认 FAST-LIO 在运行 |
| `/Odometry` 红色箭头向下倾斜 | FAST-LIO 发布的是雷达/IMU `body` 位姿，雷达倾斜安装时这是正常现象 | 不直接给导航使用；导航使用 `/Odometry_base_link` |
| `/Odometry_base_link` 没有数据 | `odometry_to_base_link` 没启动，或缺少 `body -> base_link` TF | 先启动 `mapping_auto_tf.launch.py`，再启动 `navigation.launch.py`，并检查 `ros2 run tf2_ros tf2_echo body base_link` |
| 导航姿态仍然倾斜 | 导航节点仍订阅了原始 `/Odometry` 或没有 source 新的 install | `source ~/C206Go2/install/setup.bash` 后检查 `ros2 param get /motion_controller odom_topic` |
